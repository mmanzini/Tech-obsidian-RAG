# Google Stitch

Google Labs' AI-powered UI/UX design tool that converts natural language prompts, sketches and screenshots into high-fidelity UIs. Runs on Gemini 2.5 (Flash and Pro). Free at [stitch.withgoogle.com](https://stitch.withgoogle.com) during experimental phase.

## What It Is

- An "AI-native design canvas" — design generated by AI as primary output, not as an enhancement to a human-driven tool
- Users communicate intent through text, voice or visual references; Stitch translates into complete styled UIs
- Google calls this "vibe design": intent-based creation where users describe feel and function

## Target Audience

- Solo builders, indie developers, startups with limited design resources
- PMs and developers who need to prototype quickly
- Teams seeking fast initial exploration before professional design handoff

## How It Works

**Input methods:** Natural language prompts, visual references (sketches, screenshots, URLs), voice input

**Generation flow:** Input → reads Design.md (if present) → Gemini generates UI → HTML/CSS with Tailwind → export to Figma or iterate

**Key capabilities:**
- ~45 seconds per screen; ~3 minutes for a 5-screen flow
- [[what-is-design-md|Design.md]] integration: encode system once, all generations inherit constraints
- Figma export (Standard Mode only)
- URL extraction: capture design systems from any website
- Two model tiers: Flash (350 gen/month, fast) and Pro (50 gen/month, higher quality)
- MCP integration for programmatic access from Claude Code, Cursor, etc.

## Design Philosophy

- **Intent over pixels** — express what you want, not element-by-element construction
- **Constraint-driven consistency** — Design.md encodes explicit rules applied mechanically
- **Diverge then converge** — rapid multi-direction exploration, then refinement
- **System separated from instance** — update design rules once, all future generations inherit

## Limitations

- **Static, non-responsive layouts** — limited layout structures, no pixel-perfect control
- **Minimal editing** — cannot select/modify individual elements; must regenerate
- **HTML/CSS only** — no React, Vue or framework-specific exports
- **Accessibility failures** — consistently fails WCAG contrast, touch targets, semantic HTML, ARIA (see [[accessibility-and-ai-design]])
- **No collaboration** — no multiplayer editing, comments, version history
- **Uncertain viability** — Google Labs experiment with no announced pricing; may be commercialised, integrated or sunset

## Stitch vs Figma

- **Stitch** excels at exploration — fast ideation, variant generation, visual brainstorming
- **Figma** handles everything after — collaboration, refinement, responsive design, component libraries, developer handoff
- Practical positioning: Stitch is a **pre-Figma tool**

## MCP Integration

Stitch's MCP integration enables programmatic design generation:

1. Developer describes a feature in Claude Code
2. Claude calls Stitch via MCP to generate UI
3. Stitch returns design + code
4. Claude integrates generated code into the project

Multiple open-source MCP servers wrap Stitch's API with 25+ tools. API stability is still evolving.

## Key Takeaways

- Stitch is for rapid ideation, not production design — always hand off to Figma for refinement
- Design.md integration is its most distinctive feature: encode once, apply everywhere
- Accessibility review is mandatory before any production use
- MCP integration enables design-to-code pipelines but is still maturing
- Long-term viability is uncertain as a Google Labs experiment
